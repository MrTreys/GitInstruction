# Команды Git
## Важное примечание
Все команды git соответственно должны начинаться с "git".
```sh
git <command>
```

## Команда смены директории
```sh
cd c:\folder_name
```

## Команда отображения текущей директории (MacOS, Linux)
```sh
pwd
``` 

## Листинг текущей дирректории
* Windows
```sh
dir
```

* MacOs, Linux
```sh
ls
```

## Удаление файла 
* Windows
```sh
del <filename>
```

* MacOs, Linux
```sh
rm <filename>
```

## Просмотр версий файлов
```sh
log (--oneline)
```

## Разница между основным и текужим сохранением
```sh
diff <Safe1> <Safe2>
```

## Ветки
* Просмотр веток
```sh
branch 
```

*  Добавление новой ветки
```sh
branch <New_branch_name>
```
* Удаление ветки
```sh
branch -d <Branch_name>
```

# Markdown

## Текстовые выделения

* *Курсив*
```sh
Выделяется "*" или "_" с обеих сторон.
```

* __Полужирный__
```sh
Выделяется "**" или "_" с обеих сторон.
```
## Списки
### Ненумерованный

>* Пример 1;
>* Пример 2;
>* Пример 3.

```sh 
Перечисляется через "*".
```

### Нумерованный

>1. Пример 1;
>2. Пример 2;
>3. Пример 3.

```sh 
Перечисляется через порядковые числа.
```

# GitHub

## Соединение локального и удалённого репозиториев

1. Создать аккаунт GitHub;
2. Создать локальный репозиторий;
3. Совместить локальный и  удалённый репозиторий (GitHub подскажет).

## Экспорт
```sh
push -u <name> main
```

## Импорт
```sh
pull
```

## Pull requests

```sh
1. Сделать fork репозитория;
2. clone <repository_link>;
3. branch <branch_name>;
4. checkout <branch_name>;
5. Внести изменения;
6. push;
7. Compare & pull request.
